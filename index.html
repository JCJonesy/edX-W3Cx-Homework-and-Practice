Attributes

Attributes offer additional information about an element. For example, consider a human being as an element. Humans can have countless attributes to further define them. They can have a specific name as with the id attribute. While humans can share the same name with others, element id's must be specific to only one element per file.

Humans can also be classified in general terms like male or female as with the class attribute. HTML elements can belong to more than class; and, more than one element can be in the same class.

All elements can have attributes. But, not all attributes can be used with all elements. Some attributes are specific to just one element. Others are used with particular types of elements.

Furthermore, grammar allows some descriptive attributes to be verbs - the running horse; the cheering fans. HTML offers Event Attributes in much the same way. Perhaps there is a better analogy! The grammatical version simply defines an object by it's observed event; but, the HTML event attribute is actually a user event (like clicking on an element object) that causes another event via javascript.

Special Characters

Character References are code for characters you wish to type rather than using the keyboard character directly. For example, suppose you want <p> to appear as text on your page. Typing that directly will result in creating a paragraph element rather than displaying it on the screen. For such a special case, use character references - code that replaces the keyboard character. Therefore, &lt;p&gt; in code would display <p> in a browser.
Something to think about: To display &lt;p&gt; on the screen, the code was: &amp;lt;p&amp;gt; .

There are 5 special characters that should always use entities:
CHARACTER ENTITY NAME ENTITY NUMBER
< &lt; or &LT; &#60; or &#x0003C;
> &gt; or &GT; &#62; or &#xooo3E
& &amp; or &AMP; &#38; or &#x00026;
nbsp; &nbsp; &#160; or &#x000A0;
" &quot; or &QUOT; &#34; or &#x00022;

Of course there are times when using entities is your only option. If the keyboard key combination to produce the desired character is unknown or does not exist, entities are the way to go. To see the possibilities, visit:
